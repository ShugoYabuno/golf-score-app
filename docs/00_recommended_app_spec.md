# Recommended App Spec

## Product Definition
このアプリは、ラウンド中に最短で記録できて、ラウンド後には自分の傾向を比較しやすいゴルフスコア記録アプリとして設計する。

MVPで約束する価値は次の2つに絞る。
- 1ホールの記録を10秒前後で終えられること
- 18H / 9H / ショートを混ぜずに、自分の変化を比較できること

## Target User
- 自分のラウンドを継続的に振り返りたい個人ゴルファー
- スコアだけでなく、FW / GIR / パット / ペナルティの傾向も見たい人
- 多機能なゴルフアプリより、入力の速さと見返しやすさを重視する人

## Non-Goals
- SNS / ランキング / コミュニティ
- ショット単位のトラッキング
- クラブ別詳細分析
- MVP時点でのアカウント同期や複雑なクラウド機能

## Product Decisions
1. MVPは `入力` `履歴` `分析` `設定` の4領域で構成する
2. データはローカルファーストで保持し、未完了ラウンドの再開を最優先する
3. Round開始時に記録項目を確定し、そのラウンド中は原則変更しない
4. 比較指標は round_type ごとに完全分離する
5. 欠損データは許容するが、分析では必ず対象数を併記する

## Navigation
- Homeは置かず、起点は `履歴` または `入力中ラウンド再開`
- タブ構成
  - 入力中ラウンドがある場合: `入力` `履歴` `分析` `設定`
  - 入力中ラウンドがない場合: `履歴` `分析` `設定` + `ラウンドを開始`

## Core User Flow
1. ラウンドを作成する
2. 1ホールずつ記録する
3. 中断しても後で再開する
4. ラウンド完了後に履歴へ保存する
5. 分析で平均 / ベスト / 最近の傾向を見る

## Screen Spec

### 1. Round作成
目的: 記録を始める前に、比較を壊さない最低限の前提を確定する

表示項目:
- ラウンド種別: `18H` `9H` `ショート`
- プレー日: 初期値は当日
- コース名: 任意
- Par設定: 種別に応じたテンプレートを初期値として保持し、必要時のみ編集
- 記録項目: `FWキープ` `GIR` `OB` `バンカーIN` `パット不明を許可`

仕様:
- 設定画面の値は「次の新規ラウンドの初期値」として使う
- ラウンド開始後は `config_snapshot` と `hole_pars` を固定する
- ParはコースDBに依存せず、MVPではテンプレートベースで開始する

### 2. Hole入力
目的: 立ち止まらずに、最少タップで1ホール記録する

表示項目:
- Header: `前へ` `Hole x / y` `Par`
- 進捗バー
- 必須項目
  - 打数
  - パット
  - ペナルティ合計
- 任意項目
  - FWキープ: Par4/5かつラウンド設定で有効なときのみ表示
  - GIR: ラウンド設定で有効なときのみ表示
  - OB: ラウンド設定で有効なときのみ表示
  - バンカーIN: ラウンド設定で有効なときのみ表示
- Bottom actions
  - Undo
  - 次のホールへ

仕様:
- 打数は stepper、パットとペナルティは chip で入力する
- ペナルティ合計の初期値は `0` とし、無罰打ホールでは追加操作を不要にする
- `OB` を有効にしたラウンドでは `OB回数 + その他ペナ` から `penalty_total` を自動計算する
- `パット不明を許可` がONのときは `不明` を選べる
- 各入力は都度自動保存する
- `次のホールへ` は必須が埋まったときだけ有効
- 遷移後3秒だけ Undo を有効にする
- ラウンド中の設定変更は不可とし、必要なら「このラウンドの記録項目」を参照表示するだけにする

### 3. 履歴
目的: 過去ラウンドを素早く見つけ、比較対象を選びやすくする

表示項目:
- セクション
  - 入力中ラウンド
  - 完了ラウンド
- ラウンドカード
  - 日付
  - コース名
  - round_type
  - 総打数
  - ステータス
  - 最終更新時刻

仕様:
- `入力中` は履歴の先頭に固定表示する
- 並び順は `played_at desc`
- フィルタは MVPでは `種別` と `期間` を優先する

### 4. 分析
目的: 完璧な分析ではなく、自分の変化を誤解なく見る

表示項目:
- round_type 切替: `18H` `9H` `ショート`
- 主要カード
  - ベスト
  - 平均スコア
  - 最近5ラウンド平均
  - 平均パット
  - GIR率
  - FWキープ率
  - ペナルティ平均

仕様:
- 集計対象は `COMPLETED` ラウンドのみ
- round_type をまたぐ比較はしない
- 欠損を含む指標は対象数を必ず表示する
  - 例: `GIR率 41% (対象17H)`
- Par3では FW率の対象から除外する
- `ショート` ラウンドでは FW率カードを非表示でもよい

### 5. 設定
目的: 入力画面を個人に合わせて軽くする

表示項目:
- 新規ラウンドの初期記録項目
  - FWキープ
  - GIR
  - OB
  - バンカーIN
  - パット不明を許可

仕様:
- ここで変えた設定は次回以降のラウンド開始時にだけ反映する
- 進行中ラウンドへは自動反映しない

## Round Lifecycle
- `IN_PROGRESS`
  - 1ホール以上の記録が始まったラウンド
- `COMPLETED`
  - 最終ホールまで必要項目が埋まったラウンド
- `ABANDONED`
  - 途中終了だが履歴上は保持したいラウンド

補助ルール:
- 進行中ラウンドは常に1件だけ許可する
- 再開時は最後に編集したホールを開く
- 完了後は簡易サマリーを表示して履歴へ戻す

## Data and Analytics Rules
- ベストスコアは `18H` `9H` `ショート` で分離する
- 換算スコアは将来追加しても参考値扱いに留め、ベスト計算には使わない
- 欠損があるホールでもラウンド自体は保存できるが、分析では指標ごとに対象母数を分ける
- optional項目がOFFのラウンドは、その指標の比較対象に含めない

## Implementation Priority
1. Round作成
2. Hole入力 + 自動保存 + Undo
3. 入力中ラウンド再開
4. 履歴
5. 分析
6. 設定
